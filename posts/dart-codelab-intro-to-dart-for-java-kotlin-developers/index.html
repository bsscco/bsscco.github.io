소개

이 코드랩에서 배울 것들

  • 생성자를 어떻게 만드나
  • 파라메터를 기술하는 다른 방법들
  • getter와 setter를 언제 어떻게 만드나
  • Dart는 프라이버시를 어떻게 다루나
  • 팩토리를 어떻게 만드나
  • Dart에서 함수형 프로그래밍을 어떻게 하나
  • 기타 코어 Dart 특징들

준비물

모든 예제들을 브라우저 기반 툴인 DartPad에서 작성하고 실행할 수 있습니다. IDE를 사용하려면 IntelliJ의 Dart 플로그인이나 VS Code의 Dart Code 익스텐션을 사용할 수 있습니다.

간단한 Dart 클래스 만들기

자바 튜토리얼인 Bicycle class와 같은 기능을 가진 간단한 Dart 클래스를 만들 겁니다. Bicycle 클래스는 몇 개의 private 인스턴스 변수와 getter, setter들을 가집니다. main() 메소드는 Bicycle을 인스턴스화 하고 콘솔에 출력합니다.

DartPad 열기

Bicycle 클래스 정의하기

// dart
class Bicycle {
  int cadence;
  int speed;
  int gear;
}

void main() {
  
}

// kotlin
class Bicycle(var cadence: Int, var speed: Int, var gear: Int)

fun main {

}

들여다보기

  • Dart의 main 메소드는 main() 또는 main(<String> args)으로 이름짓습니다.
  • main() 메소드는 최상위 레벨에서 살아갑니다. Dart에서는 클래스 밖 범위에 코드를 정의할 수 있습니다.
  • Dart에서는 private 키워드를 사용하지 않습니다. 프라이버시에 대한 더 자세한 내용은 “Step 3: 읽기전용 변수 추가하기” 에서 배울 겁니다.
  • main() 뿐만 아니라 Bicycle도 public으로 선언하지 않습니다. 모든 식별자는 기본적으로 public이기 때문입니다. Dart에는 public, private, protected 키워드가 없습니다.
  • Dart는 4문자 들여쓰기 대신에 2문자 들여쓰기 컨벤션을 사용합니다. dartfmt 도구를 사용하면 공백 컨벤션에 대해 걱정할 필요가 없습니다.

Bicycle 생성자 정의하기

// dart
class Bicycle {
  int cadence;
  int speed;
  int gear;

  Bicycle(this.cadence, this.speed, this.gear);
}

// kotlin
class Bicycle(var cadence: Int, var speed: Int, var gear: Int)

들여다보기

  • 이 생성자는 몸체가 없지만 Dart에서는 유효합니다.
  • no-body 생성자의 끝에는 세미콜론(;)이 붙어야 합니다.
  • 생성자 파라메터 목록에서 this를 사용하는 것은 파라메터 값들을 인스턴스 변수들에 할당하는 간결한 방법입니다.
  • 다음 코드는 위와 동일합니다.
// dart
Bicycle(int cadence, int speed, int gear) {
  this.cadence = cadence;
  this.speed = speed;
  this.gear = gear;
}

Bicycle 인스턴스를 만들고 출력하기

// dart
void main() {
  final bike = Bicycle(2, 0, 1);
  print(bike);
}

// kotlin
fun main() {
  val bike = Bicycle(2, 0, 1)
  print(bike)
}

출력 향상시키기

// dart
@override 
String toString() => 'Bicycle: $speed mph';

// kotlin
override fun toString() = "Bicycle: $speed mph"

들여다보기

  • @override 어노테이션은 분석기에게 오버라이딩 의도를 말해줍니다.
  • Dart는 문자열 리터럴을 기술할 때 '’ 또는 "” 를 지원합니다.
  • 문자열 리터럴 안에 표현식 값을 넣을 때는 문자열 interpolation을 사용하세요: ${표현식}. 만약 표현식이 식별자라면 중괄호{}를 없애도 됩니다: $vaiableName.
  • 한줄 함수 또는 메소드로 축약시킬 때는 arrow(=>) 표기법을 사용하세요.

일기전용 변수 추가하기

Dart 식별자를 private으로 만들려면 이름의 시작을 언더스코어(_)로 하면 됩니다.

speed를 private, 읽기전용으로 만들기

// dart
class Bicycle {
  int cadence;
  int _speed = 0;
  int gear;
  
  Bicycle(this.cadence, this.gear);
  
  int get speed => _speed;

  ...
}

// kotlin
class Bicycle(var cadence: Int, var gear: Int) {
  private var speed = 0
    get() = speed

  ...
}

들여다보기

  • 초기화되지 않은 변수는 null 값을 가집니다.
  • Dart 컴파일러는 언더스코어 prefix가 붙은 식별자에게 프라이버시를 적용합니다.
  • 기본적으로 Dart는 모든 public 인스턴스 변수에 암시적으로 getter, setter를 제공합니다.
  • getter, setter가 제공되기 때문에 bike.cadence, bike.gear로 접근할 수 있습니다.
